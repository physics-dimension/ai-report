Why HTML

Markdown 是源材料，HTML 是阅读界面

Claude Code 团队成员 Thariq Shihipar 在《The Unreasonable Effectiveness of HTML》中提出：当 AI 能生成页面、图表、交互和样式时，复杂输出不必再被压平成 Markdown。Markdown 仍适合记录和版本控制；HTML 更适合承载导航、图片、对比视图和交互式报告。

01

复杂信息需要空间布局

测试结果、图片对比和结论不是线性文本，HTML 可以用卡片、分栏和视觉层级降低阅读成本。

02

AI 输出不该是一堵文字墙

同样内容可以变成目录、折叠面板、图像 gallery 和证据链，而不是长篇列表。

03

交互让人保留控制感

图片放大、锚点导航、细节折叠让读者按需探索，而不是被迫从头读到尾。

04

HTML 适合最终交付

GitHub Pages 一键分享，Telegram 里只需要一个链接，详细内容在网页里看。

Executive Summary

关键发现

GPT Image 2 对短 prompt 已经很强；这个 skill 的价值不是简单“让图更漂亮”，而是把输出变得更可控、更可复用、更适合生产流程。

A

原始 prompt 已经很强

模型会主动补全文案、版式、品牌感和广告结构，有时直接生成完整海报。

B

优化 prompt 提升控制力

明确是否要文字、是否留白、是否禁止额外 logo/人物/多余产品。

C

优化图更像生产素材

更适合后期排版、品牌替换、模板化和批量生产，而不只是一次性成品。

D

文字图收益更明显

信息图、标题封面等场景中，结构化 prompt 能降低文字失控风险。

案例对比

每个案例中，左侧为原始描述直接生成，右侧为经过 `gpt-image-2-workflows` 优化后的描述生成。

Case 01

护肤品电商广告图

优化版更适合作为无文字品牌底图；原始版更像自动生成的完整广告海报。

E-commerce HeroNo TextNegative Space

原始描述

生成一张高端护肤精华的广告图，适合电商首页。

优化策略

  • 明确电商 landing page hero image。
  • 指定磨砂玻璃精华瓶、暖米色摄影棚、柔和晨光。
  • 要求无文字、无 logo、无人物、保留后期文案空间。

观察结论

原始图很完整，但会自动加品牌和卖点；优化图更干净、可控，更适合作为品牌可复用素材。
查看优化 prompt 摘要
Purpose: Create a premium skincare serum hero image for an e-commerce landing page.
Scene: Warm beige studio background with soft morning light.
Subject: One frosted glass serum bottle centered on a natural stone pedestal.
Composition: Landscape website hero banner, product centered slightly lower, lots of negative space.
Text Requirements: No text.
Constraints: No watermark. No logo. No extra bottles. No hands. No people.
Case 02

小红书发型封面

优化版更强调平台、标题和版式一致性；同时也暴露出需要后续视觉质检的 AI 手部瑕疵。

Social CoverMobile-firstTitle Control

原始描述

做一张小红书封面，主题是普通女生如何找到适合自己的发型。

优化策略

  • 明确小红书移动端封面。
  • 指定标题、副标题、粉色奶油色调。
  • 要求中心人物 + 发型对比卡片，限制额外文字。

观察结论

原始描述已能生成成熟封面；优化描述更适合模板化产品，但需要对手部和文字继续做 vision 质检。
Case 03

AI 入门信息图

文字密集型任务中，结构化 prompt 的收益最明显：文字更少、更准，布局更稳定。

InfographicText Control2×2 Structure

原始描述

做一张关于如何开始学习 AI 的信息图。

优化策略

  • 明确正方形信息图和 2×2 结构。
  • 限定标题和四个栏目文字。
  • 强调大字号、少文字、无额外文字。

观察结论

原始图内容更丰富，但更容易失控；优化图更像课程封面或社媒收藏图，文字和版式可控性更高。
Case 04

餐饮菜单牛肉面图片

优化 prompt 会让画面更生产友好：无文字、留白明确、适合菜单后期排版。

Food PhotographyCopy SpaceNo Text

原始描述

生成一张适合餐厅菜单的牛肉面图片，看起来好吃。

优化策略

  • 明确 restaurant menu hero photo。
  • 指定食材、拍摄角度、木桌、蒸汽和留白。
  • 要求无文字、无手、无多余菜品。

观察结论

原始图更像完整营销海报；优化图更适合作为菜单或外卖平台的可编辑摄影素材。
Conclusion

这个 Skill 真正提升了什么？

不是简单把 prompt 写长，而是把生成任务从“让模型自由发挥”变成“面向商业生产的可控工作流”。

可控性

明确文字、主体、构图和限制。

复用性

输出更像生产素材。

模板化

同类需求稳定复现结构。

后期友好

减少乱加 logo、文案和元素。

产品化

适合 SaaS：一句话 → 模板 → 可控图。
