Key Findings

4 notes

这部分替代传统报告里的长段“摘要”。每张卡只保留一个可执行判断。

Raw 更像自动成品
GPT Image 2 会主动补标题、卖点、logo 和排版；第一眼完整，但项目里不可控。
risk: over-posterized
Workflow 更像生产素材
通过 no text、no logo、copy space 和用途声明，输出更适合后期排版。
mode: editable asset
中文小字最危险
模型自动生成的中文、伪 logo、说明小字容易乱码；这些不应进入最终商用稿。
failure: gibberish text
文字回到 HTML / Figma
让模型产出视觉素材；让可控排版系统负责准确文字，是更稳的商业流程。
workflow: image → layout

Failure Modes

乱码 / 不可读小字
海报、信息图、小红书封面中最常见。越小的字越不可信。
inspect before publishing
伪 logo / 伪品牌
原始 prompt 容易生成不存在的品牌标识，真实项目中必须替换。
remove fake branding
过度海报化
看起来像成品，但后期难改，也很难对齐真实品牌规范。
prefer clean base image

human QA required

01

护肤品电商广告图

目标：测试是否能生成适合后期品牌排版的高端商品视觉。

E-commerceOptimized recommendedRaw adds copy
Raw Promptposterized

完成度高，但自动加入品牌、文案和卖点，真实项目中不可控。

Optimized Prompteditable base

无文字、留白充分，更适合后期添加真实品牌与文案。

原始描述

生成一张高端护肤精华的广告图，适合电商首页。

优化策略

  • 明确 landing page hero image。
  • 要求 no text / no logo。
  • 保留 copy space，避免模型乱加卖点。

结论

优化版不一定更热闹，但更适合真实商业流程。
02

小红书发型封面

目标：测试社媒封面标题、风格和移动端构图的可控性。

Social coverQA neededhands risk
Raw Promptauto-complete

模型自动理解小红书风格，标题和版面较成熟。

Optimized Promptmore controlled

更贴近指定结构，但人物手部仍需要人工或 vision 质检。

原始描述

做一张小红书封面，主题是普通女生如何找到适合自己的发型。

优化策略

  • 明确移动端封面。
  • 指定主标题、副标题和粉色系。
  • 限制额外文字。

结论

优化 prompt 提升一致性，但人物类仍不能跳过视觉质检。
03

AI 入门信息图

目标：测试文字型视觉内容是否能通过结构化 prompt 降低失控风险。

Infographictext riskstructure helps
Raw Promptdense text

内容丰富，但文字密度更高，小字和细节错误风险更大。

Optimized Promptclearer grid

2×2 结构更清晰，文字更少，更适合作为模板方向。

原始描述

做一张关于如何开始学习 AI 的信息图。

优化策略

  • 明确 2×2 模块。
  • 限制只使用指定标题和四个短句。
  • 强调 large readable text。

结论

信息图更推荐：图像生成背景/图标，文字用 HTML 或设计工具排版。
04

餐饮菜单牛肉面图片

目标：测试餐饮图是生成完整海报，还是生成可后期排版的菜单摄影素材。

Food photocopy spaceRaw adds text
Raw Promptfull poster

带标题、卖点和图标，像最终海报，但文字仍可能不可控。

Optimized Promptphoto asset

无文字、左侧留白，更适合菜单、外卖平台或后期设计。

原始描述

生成一张适合餐厅菜单的牛肉面图片，看起来好吃。

优化策略

  • 明确 restaurant menu hero photo。
  • 要求 no text，保留一侧留白。
  • 限制无手、无多余菜品。

结论

商业生产中，推荐先生成无文字素材，再用可控排版系统加字。

05

Recommendation

1 rule

把 GPT Image 2 当作视觉素材生成器，而不是最终海报排版器。

越是需要准确文字、品牌合规和可编辑性，越应该让文字和布局回到 HTML/CSS、Figma 或后期设计工具里。图片里出现 AI 自动生成的中文小字、伪 logo 或未经验证的功效数据时，不应直接商用。
